### [Git Commit Convention](https://doublesprogramming.tistory.com/256)

| Type | 설명 |
| --- | --- |
| feat: | 새로운 기능 추가 |
| fix: | 버그 수정 또는 typo |
| refactor: | 리팩토링 |
| design: | CSS 등 사용자 UI 디자인 변경 |
| comment: | 필요한 주석 추가 및 변경 |
| style: | 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우 |
| test: | 테스트(테스트 코드 추가, 수정, 삭제, 비즈니스 로직에 변경이 없는 경우) |
| chore: | 위에 걸리지 않는 기타 변경사항(빌드 스크립트 수정, assets image, 패키지 매니저 등) |
| init: | 프로젝트 초기 생성 |
| rename: | 파일 혹은 폴더명 수정하거나 옮기는 경우 |
| remove: | 파일을 삭제하는 작업만 수행하는 경우 |
| docs: | 문서 |

```
제목은 최대 50글자까지 아래에 작성: ex) Feat: 로그인 기능 구현
꼭 필요하다면 본문은 아래에 작성

# ------------------  
#   제목 첫 글자를 대문자로  
#   제목은 개조식으로 쓰기
#   제목 끝에 마침표(.) 금지  
#   제목과 본문을 한 줄 띄워 분리하기  
#   본문은 "어떻게" 보다 "무엇을", "왜"를 설명한다.  
#   본문에 여러줄의 메시지를 작성할 땐 "-"로 구분  
# ------------------
```

```markdown
# 프로젝트용 리액트 코딩 컨벤션 및 구조 정의

## 구조 및 디렉토리 정의

- 재사용 가능한 jsx 요소 -> Components
- 재사용 가능한 상태 관리 -> hooks 디렉토리에 커스텀
- 실제로 보여지는 페이지 -> pages
- Redux -> reducers(state), store
- 라우팅 -> router
- axios를 사용한 ajax 함수 -> api
- css 라이브러리 : tailwind(tailwind.config.js, index.css)

## 개선한 점

- components 파일명 -> PascalCase
- pages 파일명 -> camelCase
- js, jsx 확장자 명시화
- nvm을 통한 버전 관리(node 16)

## 개선 가능한 부분

- CRA -> Vite
- custom Hooks 사용할 상태 정의(user, session)
- 로그인 기능 추가 시 유저 인증 단계 정하기(router단계? page단계?)

```
